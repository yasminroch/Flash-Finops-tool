Síntese Executiva
Propomos substituir o Metabase por uma plataforma interna chamada Flash Analytics: uma aplicação web construída com FastAPI + React + DuckDB (ou BigQuery), com IA generativa embutida para geração de queries, explicação de dashboards e detecção de anomalias. A solução evolui de BI para produto de dados, reduz o custo de licenciamento em ~100% a longo prazo e entrega um MVP funcional em 5 dias de trabalho focado.

Integração com Dados Existentes

O conector de dados implementa uma interface DataSource com método execute_query(sql) → DataFrame. No MVP, DuckDB lê CSVs locais. Em produção, a mesma interface conecta ao BigQuery via google-cloud-bigquery sem alterar o restante da aplicação.

# Abstração de conector — troca DuckDB por BigQuery sem mudar nada acima
class DataSource(Protocol):
    async def execute(self, sql: str) -> pd.DataFrame: ...
    async def get_schema(self) -> SchemaInfo: ...

class DuckDBSource:
    async def execute(self, sql: str) -> pd.DataFrame:
        return self._conn.execute(sql).df()

class BigQuerySource:
    async def execute(self, sql: str) -> pd.DataFrame:
        return self._client.query(sql).to_dataframe()

Produto de BI — Além de Dashboards

A diferença entre uma ferramenta de BI e um produto de dados é a mesma entre um relatório e uma decisão. O Flash Analytics não entrega gráficos — entrega respostas, recomendações e autonomia para o usuário explorar sem depender de um analista.

Princípio de Design
"Não é dashboard, é produto." O dashboard é o artefato mais baixo na cadeia de valor. O produto é o que o usuário consegue fazer com os dados sem saber escrever SQL.

1. Dashboards Interativos

Diferente do Metabase, os dashboards são componentes React renderizados client-side com dados em cache. Isso permite:

  • Filtros com cross-filtering automático entre gráficos
  • Drill-down direto no gráfico sem nova página
  • Export para PNG/CSV/Excel sem plugin
  • Modo de apresentação (fullscreen, tema escuro)

2. Exploração Self-Service

Um usuário de negócio seleciona tabelas, dimensões e métricas via drag-and-drop em um explorador visual. Internamente, isso gera SQL que é executado no DW. Nenhuma linha de código necessária. O SQL gerado é exibido e pode ser copiado para o editor avançado.

3. Interface Conversacional com IA — o diferencial

Esta é a feature que transforma o produto. Um chat embutido no dashboard recebe perguntas em linguagem natural e retorna gráficos, tabelas e insights:

-- Usuário digita: "Quais centros de custo mais aumentaram licenças em outubro?"

-- IA interpreta o schema e gera:
SELECT
    cc.nome_centro_custo,
    SUM(m.licencas_adicionadas) AS novas_licencas,
    SUM(m.custo_adicional_prorata) AS custo_adicional
FROM fct_movimentacao_licencas m
JOIN dim_centro_custo cc ON m.centro_custo_id = cc.id
WHERE m.mes_ref = '2025-10'
  AND m.tipo = 'adicao'
GROUP BY 1
ORDER BY novas_licencas DESC
LIMIT 10

A IA usa RAG sobre os metadados do dbt (descrições de tabelas, colunas, regras de negócio documentadas) para gerar SQL preciso, contextualizado e sem alucinações de schema.

4. Geração Automática de Insights

A cada refresh de dashboard, um job assíncrono roda e envia o resultado das queries para o LLM com o prompt: "Analise esses dados e destaque as 3 mudanças mais relevantes em relação ao período anterior." O usuário vê um painel de "Insights do dia" antes de abrir qualquer gráfico.

5. Data Storytelling

Dashboards podem ter narrativas automáticas geradas pelo LLM: um parágrafo explicando o que os dados mostram, escrito em linguagem de negócio, atualizado a cada refresh. Transforma o dashboard em um briefing executivo.

03

Camada Analítica — FinOps do Metabase

O case de FinOps do Metabase é o melhor exemplo de por que a Flash precisa de um produto de dados próprio: as regras de negócio são complexas, o rateio é manual, e as perguntas sobre custo real são difíceis de responder hoje. A plataforma proposta resolve isso com modelagem adequada.

Modelagem — Star Schema

TabelaTipoDescrição
fct_licencas_metabase Fato Granularidade: 1 linha por mês de contrato. Contém licenças base, adicionais, custo calculado.
fct_movimentacao_licencas Fato Cada adição/remoção de licença com data, centro de custo e valor pró-rata.
dim_tempo Dimensão Calendário fiscal com referências de mês de competência e mês de faturamento (M+1).
dim_usuario Dimensão Usuário habilitado, último login, status (ativo/ocioso), centro de custo.
dim_centro_custo Dimensão Hierarquia de centros de custo para rateio.

Regras de Negócio Implementadas

Regra 1 — Custo Base Mensal

-- Contrato: 610 licenças fixas desde Nov/2025
-- Custo anual / 12 = custo mensal base
SELECT
    mes_ref,
    610 AS licencas_contrato,
    custo_anual_contrato / 12.0 AS custo_mensal_base,
    licencas_adicionais,
    licencas_adicionais * custo_unitario_dia * dias_restantes_mes AS custo_adicional_prorata
FROM fct_licencas_metabase

Regra 2 — Faturamento M+1 (exceto Nov/2025)

-- Competência Nov/2025 → fatura em Nov/2025 (exceção contratual)
-- Todo o restante: competência M → fatura em M+1
SELECT
    mes_competencia,
    CASE
        WHEN mes_competencia = '2025-11' THEN '2025-11'
        ELSE DATE_TRUNC('month', mes_competencia + INTERVAL '1 month')
    END AS mes_faturamento,
    custo_total
FROM fct_licencas_metabase

Regra 3 — Licenças Ociosas

-- Ocioso = habilitado há mais de 30 dias sem nenhum acesso
SELECT
    u.nome,
    u.centro_custo,
    u.data_habilitacao,
    MAX(a.data_acesso) AS ultimo_acesso,
    CURRENT_DATE - MAX(a.data_acesso) AS dias_sem_acesso,
    custo_unitario_mes AS custo_desperdicado
FROM dim_usuario u
LEFT JOIN fct_acessos a ON u.id = a.usuario_id
WHERE u.ativo = true
GROUP BY 1,2,3,5
HAVING dias_sem_acesso > 30 OR MAX(a.data_acesso) IS NULL
ORDER BY custo_desperdicado DESC

Requisitos de Plataforma

Gestão de Acesso — RBAC

RolePode fazerNão pode fazer
Admin Tudo: gerenciar usuários, conectores, secrets, permissões —
Analyst Criar dashboards, escrever SQL, publicar para grupo Gerenciar usuários, ver dados de outro CC sem permissão
Viewer Ver dashboards compartilhados, usar chat IA, exportar Criar/editar dashboards, escrever SQL
Service Acesso via API key para integração com outros sistemas UI, chat IA

Permissões são granulares a nível de recurso (dashboard específico, dataset, schema SQL). Implementado com Row-Level Security no DW para garantir isolamento de dados por centro de custo.

Governança e Auditoria

Toda ação é registrada em audit_log: quem acessou o quê, quando, qual query foi executada, qual resultado retornou. A tabela de auditoria é imutável (append-only) e indexada por usuário, recurso e timestamp. O relatório de auditoria está disponível como dashboard nativo da plataforma.

Organização de Conteúdo

Dashboards são organizados em Espaços (equivalente a pastas), com hierarquia de até 3 níveis. Cada Espaço tem permissões próprias. Dashboards podem estar em múltiplos Espaços via tags. Busca full-text sobre títulos, descrições e conteúdo dos dashboards.

Compartilhamento

  • Link público com token — compartilha com expiração, sem exigir login
  • Embed — iframe com JWT assinado para portais internos
  • Agendamento — envia PDF/PNG por email ou Slack em schedule configurável
  • Snapshot — versiona estado do dashboard para auditoria histórica

Comparativo e Roadmap de Evolução

Metabase vs Flash Analytics

Dimensão Metabase (atual) Flash Analytics (proposta)
Custo de licença Alto, por usuário, pró-rata em adições Zero (infra interna + custo de API LLM)
Customização Limitada ao que a ferramenta suporta 100% customizável — é código seu
Integração com IA Inexistente (recursos básicos em versões pagas) Nativa — chat, insights automáticos, anomalias
Controle de dados Dependência de fornecedor externo Dados e lógica 100% internos
Tempo de setup Rápido (SaaS) 5 dias para MVP, ~30 dias para prod completo
Manutenção Dependência do roadmap do Metabase Time interno evolui conforme necessidade
Produto de dados Dashboards apenas Chat, insights, alertas, storytelling
Curva inicial Baixa Requer engenharia dedicada por ~30 dias
Estimativa de Custo
Com 610 licenças Metabase Enterprise (~$500/usuário/ano = ~$305K/ano), a solução interna se paga no primeiro ano mesmo contando 3 meses de desenvolvimento dedicado de 2 engenheiros. A longo prazo, o custo é apenas infraestrutura e API de LLM — em geral 10–20x menor.
